# Filereader DataGrid Multi-Record Viewer

## Aim and Metadata
- Aim: Plan the first implementation for displaying parsed file contents in `dataGridView1`, with special care for files that contain multiple record types and hierarchical relationships.
- Creation date: 2026-05-08
- Last updated: 2026-05-08
- Scope target: `MainForm.cs`, `MainForm.Designer.cs`, `Filereader.csproj`, `tests/CLMS_20260430_1.TXT`, and referenced `PAMC.Formats.Medihelp` FileHelpers layouts.
- Current decision state: First implementation is complete for `Medihelp ERA`; large-file display performance has been improved with a master/detail grid; later styling and `Medihelp Pay Instruction` support remain deferred/open.
- Last audit: 2026-05-08; checklist decisions, implemented items, and pending implementation items were reconciled against `MainForm.cs`, `MainForm.Designer.cs`, `App.config`, `Filereader.csproj`, `tests/CLMS_20260430_1.TXT`, `PAMC.Formats.Medihelp.ERALayout.cs`, and `PAMC.Formats.Utilities.cs`.

## Discovered Context
- The app is a .NET Framework 4.7.2 WinForms project.
  - Evidence: `Filereader.csproj` has `<TargetFrameworkVersion>v4.7.2</TargetFrameworkVersion>` and references `System.Windows.Forms`.
- `MainForm` has a master `DataGridView` named `dataGridView1` and a detail `DataGridView` named `dataGridViewDetails`.
  - Evidence: `MainForm.Designer.cs` declares both grids inside `splitContainer2`.
- The form has a hardcoded `cbFormat` list with two options.
  - Evidence: `MainForm.Designer.cs` adds `"Medihelp ERA"` and `"Medihelp Pay Instruction"` to `cbFormat.Items`.
- The registry approach exists in `PAMC.Formats.Utilities.BuildRegistry()` but must not be used for this first implementation.
  - Evidence: User explicitly said, "no. that registry is an old idea. This is currently a hardcoded list".
- `btnView_Click` already contains a partial hardcoded `Medihelp ERA` parse path using `FileHelpers.MultiRecordEngine`.
  - Evidence: `MainForm.cs` creates `MultiRecordEngine` with `ERAServiceProviderHeaderRecord`, `ERAServiceProviderTrailerRecord`, `ERABatchFileTrailerRecord`, `ERADetailRecord`, and `ERABatchFileHeader`.
- Parsed `Medihelp ERA` results are displayed as a master/detail view.
  - Evidence: `MainForm.cs` assigns `var res = engine.ReadFile(txtSelectedFile.Text);`, passes `res` to `BindRecordsToGrid`, shows one row per parsed record in `dataGridView1`, and shows selected-record fields in `dataGridViewDetails`.
- Parse exceptions are displayed in the grid instead of crashing the form.
  - Evidence: `btnView_Click` catches `Exception ex` and calls `ShowGridMessage(ex.GetType().Name, ex.Message)`.
- The test file looks like a multi-record ERA-style pipe-delimited file.
  - Evidence: `tests/CLMS_20260430_1.TXT` starts with an `H|...` file header followed by many `E|...` detail-like rows.
- Empty or invalid startup path settings should leave the file tree empty.
  - Evidence: `MainForm.cs` loads `LastPath` through `LoadLastPathSetting`, clears `txtSelectedFile` in `UpdateDirectoryList`, and returns early when `txtPath.Text` is blank.
- The app config is minimal for .NET Framework startup.
  - Evidence: `App.config` contains only the `startup` supported runtime section.
- `PAMC.Formats.Medihelp.ERALayout.cs` defines multiple FileHelpers record classes.
  - Evidence: `ERABatchFileHeader`, `ERAServiceProviderHeaderRecord`, `ERADetailRecord`, `ERAServiceProviderTrailerRecord`, and `ERABatchFileTrailerRecord` are `[DelimitedRecord("|")]`.
- `ERASelector.ERAClaimSelector` selects record type by the first character of each line.
  - Evidence: `H` maps to `ERABatchFileHeader`, `1` maps to `ERAServiceProviderHeaderRecord`, `Z` maps to `ERAServiceProviderTrailerRecord`, `F` maps to `ERABatchFileTrailerRecord`, and everything else maps to `ERADetailRecord`.
