## New Issues
- [x] NI1. Handling Large Assemblies
  - Status: Resolved (2026-05-09)
  - Severity: Low
  - Issue: If `PAMC.Formats` contains hundreds of classes, the "Select Template" dropdown may be overwhelming.
  - Resolution: Added a search filter textbox above the template list in `ConfigWizardForm`.

- [x] NI2. Wizard UI Components
  - Status: Resolved (2026-05-08)
  - Severity: Medium
  - Issue: Step panels in `ConfigWizardForm` are currently empty; need controls for name, template selection, and field mapping.
  - Resolution: Added `txtName`, `clbTemplates`, and `clbFields` dynamically in `InitializeDynamicControls`.

- [x] NI3. Brittle Primary Key Filtering
  - Status: Resolved (2026-05-08)
  - Severity: Medium
  - Issue: `ApplyDetailFilter` in `MainForm.cs` assumes `SchemeClaimNumber` as the PK instead of using `FormatConfig.MasterColumns`.
  - Resolution: Updated `ApplyDetailFilter` to use `_activeConfig.MasterColumns` PK.

- [x] NI4. Hardcoded Error Schema
  - Status: Resolved (2026-05-08)
  - Severity: Medium
  - Issue: `ShowGridMessage` uses hardcoded columns that may clash with configurations not using those specific fields.
  - Resolution: Updated `ShowGridMessage` to use generic "Status/Message" columns.

- [x] NI5. Outdated Project Documentation
  - Status: Resolved (2026-05-09)
  - Severity: Low
  - Issue: `AGENTS.md` still refers to the format list as "intentionally hardcoded".
  - Resolution: Updated `AGENTS.md` to reflect the dynamic `formats.json` and Wizard-driven model.

- [x] NI6. Hardcoded PK Fallback
  - Status: Resolved (2026-05-09)
  - Severity: Medium
  - Issue: `ApplyDetailFilter` and `BuildDataSetFromConfig` still fall back to "SchemeClaimNumber" if no PK is found, which will fail for non-Medihelp formats.
  - Resolution: Updated `MainForm.cs` to use explicit `IsPrimaryKey` from configuration and handled null cases gracefully.

- [x] NI7. Missing Selector Selection in Wizard
  - Status: Resolved (2026-05-09)
  - Severity: High
  - Issue: Multi-record formats require a `RecordTypeSelector`. The wizard currently doesn't allow selecting one from `PAMC.Formats`.
  - Resolution: Added `cbSelectors` to `ConfigWizardForm` with reflective discovery of static methods matching the signature.

- [x] NI8. Brittle PK Heuristic
  - Status: Resolved (2026-05-09)
  - Severity: Medium
  - Issue: Wizard identifies PKs by checking if name contains "Number" or "ID". This should be user-selectable.
  - Resolution: Added `cbPrimaryKey` to Step 2 of the Wizard for explicit user selection.

- [x] NI9. Brittle Reflection Loading for Selectors
  - Status: Resolved (2026-05-09)
  - Severity: Low
  - Issue: `ConfigWizardForm` stores `SelectorTypeName` without assembly qualification, which may cause `Type.GetType` to fail in `MainForm`.
  - Resolution: Updated `ConfigWizardForm` to resolve the selector type and store its `AssemblyQualifiedName`.

- [x] NI10. Edit Existing Configurations
  - Status: Resolved (2026-05-09)
  - Severity: Medium
  - Issue: `MainForm` currently only supports adding new configurations; the "Manage Formats" button should allow editing the currently selected format.
  - Resolution: Updated `MainForm.btnManageFormats_Click` to pass the active config to the Wizard and update it in-place.

- [x] NI11. Field Picker UX Enhancements
  - Status: Resolved (2026-05-09)
  - Severity: Low
  - Issue: For records with many fields, the flat checklist is hard to navigate.
  - Resolution: Fields in Step 2 and 3 are now sorted alphabetically.

- [x] NI12. Modernize JSON Serialization
  - Status: Resolved (2026-05-09)
  - Severity: Low
  - Issue: Project uses `System.Web.Script.Serialization.JavaScriptSerializer`, which is legacy.
  - Resolution: Migrated to `Newtonsoft.Json` for all configuration persistence.
